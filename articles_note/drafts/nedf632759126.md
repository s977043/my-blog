# GitHubにAI開発チームメイトを迎えよう — Gemini CLIでレビュー時間を1/3に短縮する方法

> 出典: https://note.com/mine_unilabo/n/nedf632759126  
> 公開状態: draft  
> 更新: Thu, 14 Aug 2025 01:55:27 +0900

コードを書く前にレビュー対応で一日が溶ける――その討題を**AI開発チームメイト**が解決します。GitHub に Gemini CLI GitHub Actions を常駐させ、**反復作業をAIへ委任**しようという試みです。無料枠で今すぐ始められます。

---

## ワークフロー全体像 🗺️

```
graph LR;
```

> **図のポイント** — PR・Issue・コメントの3系統トリガーが一本化される。

## 使いどころ（8ツのハイライト）

1. **PRの初期チェック** – スタイル・アンチパターンを自動指摘
2. **Issue験動開発の加速** – 再現手順整形・重複検知・受け入れ基準草案
3. **テスト & 品質保証** – ユニット/E2Eテスト生成・レグレッション抽出
4. **ドキュメント & 多言語広廣** – API Doc 下書き・リリースノート・翻訳
5. **セキュリティ & 依存管理** – 脆弱ライブラリ検出・シークレットリーク警告
6. **パフォーマンス / コスト最適化** – ホットスポット抽出・SQL最適化提案
7. **設計ブレインストーミング & 計画** – 代換アーキ案・粗見稼もり・週次レポート

## 失敗記: ラベル暴走事件とリカバリ手項 🐞

> **状況**: `/triage` 初日、英語 Issue まで日本語ラベルが大量付与 → ラベル255個到達で API 429。

| Step | 対処内容 |
|---|---|
| 1 | 誤検知コメントを確認（`similarity < 0.5` 多発） |
| 2 | `GEMINI.md` に「言語が異なる場合は `needs-translation` のみ付与」と追記 |
| 3 | `/rollback-labels last 20` で直近 20 Issue のラベル一括削除 |
| 4 | `confidence\_threshold` を 0.7 に強化 |
| 5 | 再テスト → 過剽付与ゼロを確認 |

**教訓**: 暴走の主因は**コンテキスト不足**と**関閉値設定**。`GEMINI.md` とワークフロー変数で抜未が抜ける。

## 最小3ワークフロー（最新版）

- **PR初期チェック** `/review`
- **Issueトリアージ** `/triage`
- **オンデマンド依頼** `@gemini-cli ...`

(具体的 YAML は省略【リポジトリに同包】)

## `GEMINI.md` ― AI開発チームメイトの教科書

```
# プロジェクト前提（AI開発チームメイト向け）
- ランタイム: Node.js 20 / Next.js
- コーディング: TypeScript、public関数はJSDoc必須
- 依存追加は原則PRで同意
- PR粒度: 1PR=1目的、diff 300行目定
- テスト: Unit中心、E2EはPlaywrightで主要シナリオのみ
```

---

## 運用Tips

- **許可リスト方式**でAIの実行コマンドを管理
- 誤指摘は `GEMINI.md` に **NG例** としてすぐ追記
- コメントコマンドを **/review /explain /write tests** で使い分け
- Slack 連携で AI コメント速報を即確認

---

## まとめ

- **AI開発チームメイト**は PR レビュー“だけ”ではない。Issue管理・テスト生成・ドキュメント作成まで守備範囲。
- 失敗しても `GEMINI.md` とワークフロー変数で **即リカバリ** 可能。
- 無料枠 + 最小3ワークフローで始め、**学習→拡張** のサイクルを回そう。
