最近は「AIにコード直してもらう」ことも珍しくなくなってきました。
でも実際のプロジェクトでどこまでリファクタリングを任せられるのかは、まだ体感値がバラバラ。

今国は Python を題材に、GPT-5-Codex / Gemini CLI / Claude Code / Copilot CLI の4つを同じ課題にぶつけてみました。
狙いはシンプルで、AIがコードの可読性改善・設計整理・テスト追加まで本当にやれるのか？を確かめること。。
その過程で現場目線てきたクセや得意不得意を、現場目罰でまとめます。

## 比較するツールとモデル

・ OpenAI Codex
Codex を GPT-5 ベースに強化した最新版。大規模リファクタリングやコードレビュー性能が向上したと言われています。

・ **Gemini CLI**
Google 製のターミナルエージェント。超長文コンテキストを扱えるのが特徴。Python タスクでもどこまで踏み込めるか試す価値あり。

・ **Claude Code**
Anthropic の Claude 系。説明力に対応があり、リファクタリング理由を添えてくれるのが強み。

・ **Copilot CLI**
GitHub 提供。普毛の GitHub フローに自然に組み込めるのが魅力で、CLI から AI と会話できる。

## 実験の課題と流れ

課題はこんな内容に設定しました。

1. 既存の Python モジュール【3〜5ファイル】をリファクタリングして可読性と再利用性を改善
2. 新しい API エンドポイントを追加（例：`POST /process`）
3. pytest ベースで単体テストを追加、エッジケースも含む
4. テストが失敗したら修正依頼を投げ、どこまで追加できるか観察

## 評価したポイント

・ **正確性・バグ率**：テストをちゃんと通すか
・ **スピード感**：応答時間、最終完成までの所要時間

## ツール別比較のまとめ

各ツールの長手と短手を简澄にまとめます。

- **GPT-5-Codex**

  - 長手: リファクタリング性能が高く、大規模変更への対応
  - 短手: テストを壊すことがある、プロンプト調整が必要
- **Gemini CLI**

  - 長手: 長文コンテキスト処理、オープンソースで拡張性
  - 短手: 細部の型や例外処理でミスが出やすい、モデル調整が必要
- **Claude Code**

  - 長手: 説明付き出力、安定性
  - 短手: 応答速度がやや遅い、ツール連携が限定的
- **Copilot CLI**

  - 長手: GitHub連携、既存フローへの自然な統合
  - 短手: CLI操作の慣れが必要、プロンプトカスタマイズが少ない

## コード例・リファクタリングのイメージ

以下は AI によるリファクタリングの例示です。前のコードは内部ループがありましたが、AIは標準ライブラリを使って簡潔に書き直しました。

**リファクタリング前**

```
def sum_list(lst):
    total = 0
    for i in range(len(lst)):
        total += lst[i]
    return total
```

**リファクタリング後**

```
from typing import Iterable

def sum_list(numbers: Iterable[int]) -> int:
    return sum(numbers)
```

結論と評価のマトメ

今回の比較では、AIコーディングエージェントがPythonコードのリファクタリングをどこまで実現できるかを確認しました。

実験の結果、多くのツールは「基本的なリファクタリングや小覠修正」には十分実用できることがわかりました。
しかし、长文のコンテキストを含む大覧なリファクタリングには、現場のレビューが必要な場面もあると感じました。

わかりやすく整理すると、

- GPT-5-Codex…大覧のリファクタリングを高級に出力できるが、テストを壊すことがある
- Gemini CLI…長文コンテキストの保持に強みがあるが、型変換や例外処理で細かなミスが出る
- Claude Code…説明力に優れ、監督に優しいが、忍ちがちょっと遅い
- Copilot CLI…GitHub連携の自然さが強みである一方、CLI操作に慣れが必要

最後に、AIに任せられる約界と人関の必要性を明記しておきます。基本的なリファクタリングは直接AIに簡単に実践させることができますが、大要な模様警奉や保守性検証については人間が監督し、コード評価とAIの協力を不可絶にするのが重要だと感じました。
