# note記事品質チェックリスト

note記事を「書いたもの」から、読者の悩みに届き、クリックされ、最後まで読まれ、保存・購入・信頼につながるコンテンツへ近づけるためのチェックリスト。

記事作成前、執筆中、公開前レビューで使用する。すべてを満たすことが目的ではなく、記事の目的・読者・公開状態に照らして、弱い箇所を見つけるために使う。

元情報は Issue #209 の本文を初版の正とする。元画像は Issue に未添付で、画像下部に不足がある可能性があるため、必要に応じて後続で補完する。

## 使うタイミング

- 記事作成前: テーマ、読者、読後変化を確認する
- 執筆中: 本文構成、余白、具体性を確認する
- 公開前レビュー: タイトル、リード、読者体験、CTAを確認する

## テーマ設計

- [ ] 普遍的な悩みに基づいたテーマか？
- [ ] 読者の理想像に訴求しているか？
- [ ] 一時的な流行や炎上に頼っていないか？
- [ ] 書き手の専門性と連動したテーマか？
- [ ] 過去の自分に届けたい内容になっているか？
- [ ] ターゲット層の検索意図を想定しているか？
- [ ] テーマに一貫性があるか？
- [ ] 需要のあるジャンルに属しているか？
- [ ] SNSと親和性が高いか？
- [ ] 時間が経っても読まれるテーマか？

## サムネ・タイトル設計

- [ ] 読者の課題を明示しているか？
- [ ] クリックしたくなる表現になっているか？
- [ ] サムネとタイトルで内容を補完しあえているか？
- [ ] バズ狙いの釣りタイトルではないか？
- [ ] SEOキーワードが含まれているか？
- [ ] 読後の変化が想像できるか？
- [ ] コンテンツの中身と一致しているか？
- [ ] 数字や具体例が含まれているか？
- [ ] 共感・痛み・未来などを感じるか？
- [ ] 競合と被っていないか？

## 本文構成

- [ ] 初見で内容が想像できるか？
- [ ] 導入で共感または問題提起をしているか？
- [ ] 誰に向けた記事か明確か？
- [ ] 自己紹介から入っていないか？
- [ ] その記事で満たせるニーズと満たせないものを提示しているか？
- [ ] 中盤で具体的な体験やロジックがあるか？
- [ ] 結論がしっかりと提示されているか？
- [ ] 目次またはセクション分けがあるか？
- [ ] 長すぎず、短すぎないか？
- [ ] 例え話や図解の補助があるか？
- [ ] 文章が論理的につながっているか？
- [ ] 見出しごとに意図があるか？
- [ ] まとめや振り返りがあるか？
- [ ] スマホでも読みやすい構成か？
- [ ] 冒頭に更新情報を入れる必要があるか？
- [ ] 記事によって生まれた権威を提示しているか？
- [ ] 第三者の口コミを分類して提示できているか？
- [ ] 記事の信頼性・価格の妥当性を論理的に解説しているか？
- [ ] CTAが記事の目的と読者の状態に合っているか？

## 余白設計

- [ ] 全部説明して安心しているだけになっていないか？
- [ ] 読者が自分事として考えられる問いが入っているか？
- [ ] 体験談を「全部言われたオチまで」書いていないか？
- [ ] 最後を説明だけで締めていないか？

## 読者体験

- [ ] 導入に、読者の頭の中にズレを作る問いがあるか？
- [ ] 抽象的すぎてよく分からないものになっていないか？
- [ ] リリース時点から全力の状態で出しているか？
- [ ] 読者の「今の悩み」が問いで提示されているか？
- [ ] 読んで得られるベネフィットが明確か？
- [ ] 無料パートだけで「良い話だったな」で終わりそうになっていないか？
- [ ] 無料パートは論理と感情の両立ができているか？
- [ ] 読後に新たな視点を得られるか？
- [ ] 感情が動かされる構成か？
- [ ] 書き手の人間性が伝わるか？
- [ ] 最後まで読まれる工夫があるか？
- [ ] 「保存したい」と思える内容か？
- [ ] 学習体験が上がるコンテンツを用意できているか？

## 公開前の最終確認

- [ ] テーマ、タイトル、リード、CTAが同じ読者像に向いているか？
- [ ] 記事の価値が冒頭で伝わるか？
- [ ] 読者が読み終えた後に取る行動が明確か？
- [ ] noteの公開状態に応じた運用制約を満たしているか？
