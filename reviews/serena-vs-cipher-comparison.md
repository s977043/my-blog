### 該当箇所 2
L138 （curl による install.sh 実行の注意書きの位置）

```bash
# 例：uv を使う場合（公式の案内に準拠）
# ⚠️ セキュリティ注意：実行前にスクリプト内容を確認することを推奨
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 問題点
`curl | sh` パターンのセキュリティリスクへの注意書きはコードブロック内にコメントとして挿入されているが、実行時にはコメントとして無視される（読者がコピペして実行すると注意書きは画面に残らない）。また `⚠️` のような絵文字をコードブロック内に入れる形は、ターミナルでのペーストミスを誘発する可能性がある。

### 提案
注意書きはコードブロック外の `:::message alert` ブロックに出す。

````markdown
:::message alert
`curl | sh` でインストールする前に、スクリプト内容を必ず確認してください。
`curl -LsSf https://astral.sh/uv/install.sh -o install.sh` でダウンロードし、内容を確認してから `sh install.sh` で実行する方法を推奨します。
:::

```bash
# 例：uv を使う場合（公式の案内に準拠）
curl -LsSf https://astral.sh/uv/install.sh | sh

# Serena を取得
git clone https://github.com/oraios/serena.git
cd serena
```
````

### 該当箇所 3
L143-L150 （Serena の設定テンプレート名）

```bash
# 設定テンプレートを準備
cp project.template.yml my-project.yml
# ↳ 言語やプロジェクトルートを指定

cp serena_config.template.yml serena_config.yml
# ↳ 上記 my-project.yml を登録
```

### 問題点
`project.template.yml` や `serena_config.template.yml` というファイル名は、Serena 公式リポジトリの命名規則と一致していない可能性がある。Serena 公式（github.com/oraios/serena）のREADMEでは `project.template.yml` ではなく、現在は `.serena/project.yml` 形式や、`serena_config.yml` の仕組みも含めて構成が変化している可能性が高い。OSSは更新頻度が高いため、公式手順との乖離が起きやすい。

### 提案
公式リポジトリへのリンクを明示し、「執筆時点の構成」であることを注記する。最新手順は必ず公式READMEを参照するよう読者を促す。

```markdown
:::message
Serena は活発に更新されているため、設定ファイル名やディレクトリ構成は執筆時点のものです。
最新の導入手順は [公式README](https://github.com/oraios/serena#installation) を参照してください。
:::

```bash
# 設定テンプレートを準備（執筆時点の例）
cp project.template.yml my-project.yml
cp serena_config.template.yml serena_config.yml
```
```

### 該当箇所 4
L248 （curl のヘルスチェック URL）

```bash
# ヘルスチェック（タイムアウト付き）
curl -m 10 http://localhost:3000/health
```

### 問題点
Cipher のヘルスチェックエンドポイントが `/health` であること、およびデフォルトポートが 3000 であることが、記事本文中で根拠なく記載されている。Cipher（@byterover/cipher）のドキュメントや実装との整合確認がなく、読者がそのまま実行してもエラーになる可能性がある。また `cipher --mode api` のデフォルトポートも明記されていない。

### 提案
ポートとエンドポイントの根拠を注記するか、「例示のため」として明示し、実際のポート・URLは起動時のログを参照するよう促す。

```bash
# Cipher起動時のログで実ポートを確認する
cipher --mode api
# => 例: "Listening on http://localhost:3000"

# ヘルスチェック（タイムアウト付き。URLは起動ログに合わせて置き換え）
curl -m 10 http://localhost:3000/health
```

### 該当箇所 5
L256 （`cipher --mode index` コマンド）

```markdown
- **初回インデックス時間**：大規模リポジトリは待つ。最初は小さく。
  - 解決策：`cipher --mode index --path ./src/components` のように範囲を限定
```

### 問題点
`cipher --mode index` というモードが Cipher に実在するかの記載根拠がない。先行するL238の記述では `cipher` の `--mode` は `api` と `mcp` のみ例示されており、`index` モードは突然ここで登場する。公式ドキュメントに存在しないモードを提示している場合、読者が実行してエラーになる。

### 提案
公式に `index` モードが存在することを確認の上で記述するか、存在しない場合は「CLIから初回インデックス範囲を指定する代替手段」として正しいコマンドに置き換える。存在有無が不確実な場合は削除を推奨。

```markdown
- **初回インデックス時間**：大規模リポジトリは待つ。最初は小さめのディレクトリから試し、段階的に範囲を広げる。
```

## 総合評価

### 良い点
- **判断基準の明示**: 「個人×CLI中心はSerena／IDE中心・複数人・大規模はCipher」という結論が明確
- **比較軸の定義が丁寧**: 形態・文脈・連携・リソースなど、読者が気にする観点を網羅
- **プロンプト例の良し悪し**: Serena節の「良い例／悪い例」が具体的で読者が真似しやすい
- **Mermaid図・アーキテクチャ画像の活用**: 視覚的理解を助ける構成

### 改善点
- **比較表の重複**（3.2節と4章）
- **シリーズ構成宣言と本文の乖離**（前中後編が1記事に集約）
- **結論文の過剰な反復**（同じ文が7箇所）
- **Cipherのコマンド例の根拠不足**（`--mode index`、`/health` エンドポイント、デフォルトポート）
- **公式仕様との乖離リスクへの注記不足**（Serena/Cipherは更新頻度が高いOSSのため）

### 推奨アクション
1. **比較表の統合**: 3.2節の表を削除、4章の詳細表に一本化
2. **シリーズ構成の整理**: 前・中・後編宣言を削除し、Part 1/2/3 表記に変更
3. **結論の反復削減**: 中盤の繰り返しを2-3箇所減らす
4. **コマンド例の根拠確認**: 特に `cipher --mode index`、`/health`、デフォルトポートを公式で確認
5. **公式参照の明示**: Serena/Cipher ともに「執筆時点」であることを注記し、公式READMEへの導線を強化

### SEO観点での改善提案
- **タイトル最適化**: 現タイトル「2分で決まる：Serena vs Cipher　思想とアーキテクチャ——CLIオンデマンドか常駐メモリか」は情報量が多く、検索結果で後半が切れる可能性がある。「Serena vs Cipher 比較：CLI型と常駐型の使い分け（2025年版）」のようにキーワードを前方に寄せると流入が増えやすい
- **キーワード露出**: 「MCP」「AIコーディング支援」「開発文脈管理」などのキーワードは一定の検索ボリュームがある。初出時にh2/h3 見出しに含めると検索エンジンに認識されやすい
- **画像のalt属性**: Mermaidの代わりに画像を使用している部分（L83, L87, L330）の alt テキストは設定されているが、検索流入を意識するならキーワードを含めた alt に改善可能（例: "Serena CLI型アーキテクチャ：ターミナルから都度コンテキスト生成"）
- **比較記事の検索ニーズ**: 「◯◯ vs ××」クエリは比較検討段階の流入が多い。記事前半で「どちらも使った人の結論」が早い段階で読めることが重要で、現行のTL;DRは適切。さらに「導入順のおすすめ」（付録のPhase 1-4）を目次から直接ジャンプできるよう、アンカーリンクを強化すると効果的
- **内部リンク**: 関連記事セクションで同著者の他記事へリンクが張られているが、本記事の Serena/MCP 文脈と最も関連が深い `sdd-tdd-nonblocking-agent` 記事内の Serena 言及部分とはアンカー付きリンクにすると、内部リンクのキーワード価値が上がる
