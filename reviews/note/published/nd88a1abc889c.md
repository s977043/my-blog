## 指摘コメント

### 該当箇所 1
L15 JTFスタイル違反（ダッシュ「――」の使用）

```
Googleのパーパス――「世界中の情報を整理し、世界中の人がアクセスできて使えるようにすること」に強く共感していて、
```

### 問題点
JTFスタイルガイドでは全角ダッシュ（「―」「――」）を原則使用しない。引用・言い換えの前置きには句読点や括弧類を用いるのが望ましい。本記事内では L194 にも同種の表現がある。

### 提案
```
Googleのパーパス、「世界中の情報を整理し、世界中の人がアクセスできて使えるようにすること」に強く共感していて、
```
または
```
Googleのパーパス（「世界中の情報を整理し、世界中の人がアクセスできて使えるようにすること」）に強く共感していて、
```

### 該当箇所 3
L17 と L27 の重複表現（AI下書きの注記が近接して2回）

```
※本稿はAIで一部下書きを支援し、最終確認は人手で行っています。
...
※本記事ではAIで下書きを一部支援し、最終確認は人手で行っています。
```

同じ趣旨の透明性注記が12行以内に2回登場しており、リード部の流れを切ってしまう。note読者はリードで離脱判断するため、冗長な繰り返しは読了率に悪影響。L193-L194 にも同趣旨の「透明性のためのメモ」があり、実質3度出てくる。

### 提案
冒頭は1回だけに統一し、記事末の「透明性のためのメモ」に集約する。
```
（L27 の「※本記事ではAIで下書きを一部支援し、最終確認は人手で行っています。」を削除）
```

### 該当箇所 4
L67-L69 空の見出しと見出し階層の乱れ

```
### ＜ドキュメントの要点＞

### Search Essentials（Google検索セントラル）の要点
```

### 問題点
`### ＜ドキュメントの要点＞` が配下に本文を持たず、直後に同じ H3 が続いている。見出し構造としては親（見出しのまとめ）が H3 で、子も H3 となり階層が崩れている。note のモバイル目次表示でも空セクションが目立つ。

### 提案
```
### ドキュメントの要点

#### Search Essentials（Google検索セントラル）の要点
```
（親を H3 にしたうえで子を H4 に下げる。もしくは `### ＜ドキュメントの要点＞` を削除して各要点 H3 を並列に残す）

### 該当箇所 8
L127-L143 コードブロック内でタブ文字と全角中黒が混在し、コピペ利用時の体験が崩れる

```
【サイト情報】
	•	主要5URLと各ページのタイトル／ディスクリプション／H1（貼付）
	•	Search Console：過去28日のクエリTop10（貼付）
	•	Core Web Vitals：LCP／INP／CLS（数値）
```

### 問題点
プロンプトはコピペして使うことを意図しているが、Mac風の「タブ＋中黒（•）＋タブ」形式になっており、ChatGPT や Gemini 等にそのまま貼るとインデントや箇条書き解釈が不安定になる。ハウツー記事として再現可能性に直結する。

### 提案
```
【サイト情報】
- 主要5URLと各ページのタイトル／ディスクリプション／H1（貼付）
- Search Console：過去28日のクエリTop10（貼付）
- Core Web Vitals：LCP／INP／CLS（数値）

【出力してほしい内容】
1. 現状の要点（良い点／改善点）
2. 優先順位Top3（理由・所要労力：小／中／大）
3. 具体的な修正例
   - 各URLのタイトル／ディスクリプション改稿案（日本語で2案ずつ、30〜55字／80〜120字）
   - 内部リンク追加案（どの文言からどこへ）
4. 次回チェックするKPI（SCの指標名と見方）
```

### 該当箇所 9
L169 「20%前後が目安」という数値の根拠が曖昧

```
- **CTR**：メタ改善が当たると上昇（1〜3位で20%前後が目安、10位付近は2〜5%でも妥当）
```

### 問題点
CTRの目安値は業界・キーワード種別（指名/一般）によって大きく変動する。解説/ハウツー記事としての正確性観点では、出典（Advanced Web Ranking や Sistrix 等のCTR調査）を示すか、「一般的な目安」である旨の注記が必要。冒頭で「Google公式」を強調している本記事のトーンに対し、数値の根拠が弱い点が気になる。

### 提案
```
- **CTR**：メタ改善が当たると上昇（※業界・クエリ種別で変動。一般論として1〜3位で10〜30%、10位付近は2〜5%程度が観測されやすい。自サイトのSC実測を基準に変化量で見る）
```

## 総合評価

### 良い点
- 冒頭フックが相談シーンから入り、noteらしい共感導線を持っている（L12-L13）
- 「AIに任せる/人が決める」の線引きが章立てで明確で、ハウツー記事の軸がブレない（L82-L102）
- Google公式ドキュメントへのリンクが3本そろっており、一次情報への誠実な姿勢が見える（L48-L55）
- 最後に「30分クイックスタート」「1本だけ直す」という具体的最小行動が提示されており、読者の行動転換コストが低い（L154-L162, L177-L183）
- 透明性のためのAI利用注記があり、昨今のAI生成記事警戒感に配慮している（L193-L194）

### 改善点
- JTFスタイル違反（ダッシュ「――」）が2箇所ある
- 同趣旨の注記・文章が複数回登場し、リードの密度が下がっている
- 見出し階層に崩れ（空の H3、末尾の空 H2、H3 の親子関係）がある
- 用語の不統一（「抑える/押さえる」）や文脈不明の短句（「体験」）が残る
- 手順・数値の再現性確保（プロンプトの体裁、CTR目安の根拠）がハウツー記事として弱い

### 推奨アクション
1. JTFスタイル違反（L15, L194）とタイポ（L65）、空H2（L198）を優先修正
2. リード部の冗長な注記（L17 と L27）を末尾の透明性メモに集約
3. 見出し階層を整理（L67「＜ドキュメントの要点＞」の扱いを決める）
4. プロンプトブロック（L127-L143）をコピペ最適な体裁に置き換え
5. CTR目安値（L169）に業界差異の注記か出典を添える
