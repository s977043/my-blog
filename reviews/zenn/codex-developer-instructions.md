## 指摘コメント

### 該当箇所 1

L61-L69（命令階層の順序が要確認）

```text
developer_instructions
↓
system prompt
↓
AGENTS.md
↓
user input
```

### 問題点

L71-L73 で「内部実装の詳細を外部から完全に断定することはできません」と留保している点はよいが、L61-L69 の図は `developer_instructions` が `system prompt` より上位に見える。OpenAI/Codex の実際の命令優先順位は提供形態や実装で変わる可能性があり、公開記事で断定図として出すと事実誤認に見えるリスクがある。現状は「概念的には」と前置きしているが、図の強さに対して留保が後置きになっている。

### 提案

L57-L73 を「実務上のモデル」であることが先に伝わる形へ変更する。

````markdown
## ざっくりした設計モデル

この記事では、実装内部の正確な優先順位を断定するのではなく、実務上は次のように分けて考えます。

```text
上位の行動ルール: developer_instructions
プロジェクト文脈: AGENTS.md
今回の依頼: user prompt
```
````

**優先度: High**（技術正確性のブロック要因）

### 該当箇所 2

L28 / L41-L55（`developer_instructions` の導入が具体的な配置・設定方法に接続しない）

```markdown
その中核になるのが `developer_instructions` です。

`developer_instructions` は、Codex の振る舞いを制御するための上位レイヤーの指示です。
```

### 問題点

### 提案

L30 付近に前提を追加する。

```markdown
なお、`developer_instructions` の指定方法や名称は利用環境によって異なります。
この記事では特定の設定ファイル名を前提にせず、「エージェント全体に常時効かせる判断ルール」という設計単位として扱います。
```

**優先度: High**（読者の実装可能性と誤解防止）

### 問題点

**優先度: Medium**（読了導線の改善）

### 問題点

L167-L208 でテンプレート全文を提示したあと、L216-L397 で各セクションを再掲しながら説明している。長尺記事としては情報量がある一方、読者は同じコードブロックの見出しを何度も読むことになる。Zenn ではテンプレート全文を `:::details` に畳むか、先に構造表を出してから必要な章だけ詳説したほうが読みやすい。

## 総合評価

### 良い点

- Front Matter は Zenn 下書きとして問題ない
- `developer_instructions` と `AGENTS.md` の比較表（L91-L98）は、本記事の中心価値として機能している
- Safety / Diff Discipline / When Done など、実務事故に直結する判断ルールをテンプレートに含めている
- L71-L73 で内部実装を断定しない留保があり、事実誤認を避けようとしている

### 改善点

- 命令階層図（L61-L69）が強く、実装上の正確性に疑義が残る
- `developer_instructions` の配置・指定方法を特定しない前提が本文にない
- 長尺に対して冒頭の価値先出しと全体像が弱い
- テンプレート全文と詳細解説の重複で、読了負荷が高い

### 推奨アクション

1. **High 2 件（指摘 1 / 2）を先に反映**: 命令階層の断定を避け、`developer_instructions` の扱いを「設計単位」として明記する
2. **Medium 2 件（指摘 3 / 4）を反映**: 冒頭 `:::message` と構造表または `:::details` で長尺記事の導線を整える
3. **Low 2 件（指摘 5 / 6）は仕上げで調整**: 造語ラベルの扱いとアンチパターンの先出し

### SEO観点での改善提案

- `topics` は 5 個以内で、`codex`, `ai駆動開発`, `llm`, `agents-md`, `プロンプト` の組み合わせは本文テーマと概ね一致している
- タイトルに `developer_instructions` が入っていないため、検索流入を狙うならサブタイトルか冒頭で明示する
- Zenn 内検索では `AGENTS.md` / `Codex` / `developer_instructions` の語が効くため、L30-L39 の「扱うこと」は維持推奨
