### 該当箇所 1
L88-L97 （著者フィールド対応表の列順とパス表記）

```markdown
| 場所 | WordPress 一般慣例 | note 公式エクスポート |
|---|---|---|
| item `<dc:creator>` | login ID（`mine_unilabo`） | **表示名（`みね`）** |
| channel `<wp:author_display_name>` | 表示名（`みね`） | **login ID（`mine_unilabo`）** |
```

### 問題点

本記事の主題は「note の実装に合わせる」ことであり、読者が参照すべき正解は右列（note 公式エクスポート）だが、表の視線誘導（左→右）では「まず慣例を見てから note は逆」の順に読むことになる。太字で正解側を強調しているものの、列名自体の優先順位が記号的に示されておらず、「どちらに合わせれば note に通るのか」の読取コストが残る。

加えて `<wp:author_display_name>` は WXR 仕様上 `<channel>` 直下の `<wp:author>` ブロック配下の子要素であり、厳密には `channel <wp:author_display_name>` ではなく `channel > wp:author > wp:author_display_name` の入れ子。読者が自分の WXR を確認する際、この入れ子を知らないと grep しても見つからず、パス表記に齟齬が生じる。

### 提案

正解側（note 公式エクスポート）を左列に入れ替え、パスを完全形（入れ子も含む）で示す。

```markdown
| 場所 | note 公式エクスポート（これが正解） | WordPress 一般慣例（参考） |
|---|---|---|
| `<item>` 直下 `<dc:creator>` | **表示名（`みね`）** | login ID（`mine_unilabo`） |
| `<channel>` > `<wp:author>` > `<wp:author_display_name>` | **login ID（`mine_unilabo`）** | 表示名（`みね`） |
```

**優先度: Medium**

### 該当箇所 7
L21 / 記事全体 （note のエラーメッセージの正確な引用形式）

```markdown
note.com が提供する WXR（WordPress eXtended RSS）インポート機能を使って、AI エージェントが生成した記事を下書きに投入しようとしたところ、「インポートにエラーが発生したため、記事の読み込みに失敗しました」で止まりました。
```

### 問題点

L21 の冒頭でエラーメッセージ「インポートにエラーが発生したため、記事の読み込みに失敗しました」が本文に直接引用されているが、**引用ブロック / コードブロックではなく本文中の「」囲み**になっており、以降の本文（L84「読み込みを中断します」）で再掲するときにも書式が揃っていない。読者が自分のブラウザで出たエラーメッセージと記事の文字列を突き合わせたいとき、引用ブロック形式（`>` または `` ` ``）のほうが検索（Ctrl+F）とコピペの両方で拾いやすい。

また、実際の note のエラーメッセージが文末の句読点（`。`）を含むかどうかで検索ヒット率が変わるため、**note が実際に表示する文字列をそのまま転記**するのが望ましい（現状では文末の句点が記事の「。で止まりました」と連結されており、引用範囲が読者には曖昧）。

### 提案

```markdown
note.com が提供する WXR（WordPress eXtended RSS）インポート機能を使って、AI エージェントが生成した記事を下書きに投入しようとしたところ、以下のエラーで止まりました。

> インポートにエラーが発生したため、記事の読み込みに失敗しました

（※ note UI が実際に表示する文言。検索で同じエラーに悩む読者が到達できるよう、原文をそのまま転記）
```

**優先度: Low**（機能に影響はないが、SEO キーワード到達性と検索ヒット率に効く）

### 該当箇所 8
L121-L127 （`verify_wxr.py` の実装スニペットに import 行がない）

```python
def item_wp_tags(item):
    prefix = "{http://wordpress.org/export/1.2/}"
    return {
        el.tag.split("}", 1)[-1]
        for el in item
        if el.tag.startswith(prefix)
    }

ref_wp = item_wp_tags(ref_item)
gen_wp = item_wp_tags(gen_item)
missing_wp = ref_wp - gen_wp
if missing_wp:
    errors.append(f"<item> に必須 wp:* タグが欠落: {sorted(missing_wp)}")
```

### 問題点

このスニペットは読者が「`set` 差分で 14 フィールド欠落を検出する設計」を理解するための抜粋だが、**`ref_item` / `gen_item` / `errors` 変数の出所が示されていない**。記事の流れだけ見ると、読者が自分のコードに貼り付けようとした場合に「これらは何？」で詰まる。L131 の補足で「参照エクスポートが手元にあるなら、スキーマ定義書より実物比較のほうが信頼性が高い」と結論しているが、スニペットとしての完結性に欠ける。

また、名前空間 URI `{http://wordpress.org/export/1.2/}` はハードコードされており、WXR 1.0 / 1.1 のレガシー環境では prefix が変わることへの注意喚起が無い（note はおそらく 1.2 で統一されているため実害は少ないが、`wp:*` を汎用的に抜き出す記事として注記があると親切）。

### 提案

スニペットに最小限のコンテキスト（変数の出所）をコメントで補う。

```python
# ref_item / gen_item は、それぞれ公式エクスポートと生成WXRの
# <item> 要素（ElementTree.Element）を1件ずつ取り出したもの。
# errors は検出結果を溜める list[str]。

def item_wp_tags(item):
    # WXR 1.2 の wp 名前空間。note のエクスポートはこれで固定。
    prefix = "{http://wordpress.org/export/1.2/}"
    return {
        el.tag.split("}", 1)[-1]
        for el in item
        if el.tag.startswith(prefix)
    }

ref_wp = item_wp_tags(ref_item)
gen_wp = item_wp_tags(gen_item)
missing_wp = ref_wp - gen_wp
if missing_wp:
    errors.append(f"<item> に必須 wp:* タグが欠落: {sorted(missing_wp)}")
```

**優先度: Low**（抜粋スニペットの完結性向上）

### 該当箇所 9
L117-L119 （チェック項目 4 の判定条件が曖昧）

```markdown
4. `<dc:creator>` が小文字英数 + アンダースコアのみ（= login ID っぽい）なら警告
5. `<img src>` が `https://` で始まっているか（note は相対パス / `http://` いずれも取り込み不可）
```

### 問題点

項目 4「小文字英数 + アンダースコアのみ（= login ID っぽい）」はヒューリスティックな判定だが、**日本語表示名を使う note ユーザー（例: `みね`、`たろう`）以外の、ローマ字表示名（例: `Taro`, `MineT`）の場合に誤警告が出る** 可能性について触れていない。技術的に正確さを重視するなら「日本語文字を含まない場合に警告」という条件のほうが偽陽性 / 偽陰性の議論ができる。

項目 5 の「note は相対パス / `http://` いずれも取り込み不可」は実挙動の記述だが、**`verify_wxr.py` 自体の判定は「https で始まるか」だけ** である旨が読者には区別しにくい。スクリプトの判定ロジックと note の実挙動は別物であり、前者のほうが厳密（相対パスも `http://` もどちらも `https://` 始まりではないので一括で警告対象になる）という関係。

### 提案

警告ロジックのヒューリスティックであることを明示する。

```markdown
4. `<dc:creator>` が小文字英数 + アンダースコアのみ（= login ID っぽい）なら警告（日本語表示名を使う前提のヒューリスティック。ローマ字表示名の場合は誤検知の可能性あり）
5. `<img src>` が `https://` で始まっているか（note が取り込めるのは https のみ。相対パスや `http://` はここで弾いて事前検知）
```

**優先度: Low**（読者が自分の環境で警告を抑制 / 調整したいときの判断材料）
