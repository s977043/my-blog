```markdown
## 参考

- [Growth Lab](https://the3396.com/articles)

```

### 問題点

本文テーマは SEO 記事 / SNS 投稿 / AI による再利用だが、L284-L285 の 2 本は技術 Docs 要約と知識管理ワークフローで、本文中に接続説明がない。Zenn の関連記事として著者回遊を狙う意図は理解できるが、読者から見ると「なぜこの2本が参考なのか」が分かりにくい。publish 前には本文主題に近い関連記事へ差し替えるか、関連理由を注釈で示したほうがよい。

### 該当箇所 3

L171-L213（AIで再利用する手順が、前章までの「3つの差分」と接続しきれていない）

```markdown
## AIで再利用する手順

1本の記事からSNS投稿を作るなら、この順番が扱いやすいです。

1. 記事から主張を3つ抜き出す
2. それぞれに「読者の困りごと」を対応させる
3. SNS用にフックを作る
4. 具体例を1つだけ入れる
5. CTAを記事リンクか会話の起点にする
```

### 問題点

前半は「構成 / 文体 / CTA」の 3 差分で整理されているが、L175-L179 の手順は「主張 / 困りごと / フック / 具体例 / CTA」の 5 手順になっており、3差分との対応が明示されない。内容は実用的だが、記事タイトルの「3つの差分」から読み進めた読者にとって、後半で分類軸が変わったように見える。

### 提案

L171 直下に対応を追加する。

```markdown
ここでは、前半の3差分を次のように作業へ落とします。

- 構成: 主張と読者の困りごとを分ける
- 文体: フックと具体例をSNS向けに作り直す
- CTA: 記事リンクか会話の起点に分ける
```

**優先度: Medium**（構成の接続改善）

### 該当箇所 4

L31-L70（差分1だけ例と表が厚く、差分2/3との情報量が非対称）

```markdown
## 差分1：構成
...
| チャネル | 最初に置くもの | 読者の状態 | 失敗しやすい書き方 |
```

### 問題点

差分1は構造例、比較表、具体文例まであり非常に分かりやすい。一方、差分2は文体変換プロンプトと良い/悪い例、差分3は CTA 表があるものの、3差分全体を横断する比較表がない。現状でも読めるが、タイトルで「3つの差分」と約束しているため、最初に 3 差分の一覧表を置くとスキャン性が上がる。

### 提案

TL;DR 後に追加する。

```markdown
| 差分 | SEO記事 | SNS投稿 |
| --- | --- | --- |
| 構成 | 検索意図への回答から始める | フックから始める |
| 文体 | 正確さと再現性を優先する | 温度感と人の声を優先する |
| CTA | 次の行動へ送る | 次の接触を作る |
```

**優先度: Low**（読みやすさ改善）

### 該当箇所 5

L260-L263（SNS チャネルの列挙に Instagram が出るが、本文の出力テンプレートにはない）

```markdown
SEO記事、X、LinkedIn、Instagramでは、読者の温度も行動も違います。
同じリンク誘導を貼るより、チャネルごとに「次に何をしてほしいか」を変えたほうが自然です。
```

### 問題点

SEO記事、X、LinkedInでは、読者の温度も行動も違います。

または Instagram も扱うなら、L233-L237 のテンプレートに `Instagram向けキャプション` を追加する。

## 総合評価

### 良い点

- Front Matter は Zenn 下書きとして問題ない
- 冒頭で SEO と SNS の読者状態の違いを説明しており、タイトルの約束と本文が一致している
- 差分1の比較表、差分2の悪い例/良い例、差分3の CTA 表は実用性が高い
- プロンプト例が「条件」「出力」を明示しており、読者がそのまま試しやすい

### 改善点

- 参考リンクのうち 2 本が本文主題と弱く、publish 前に差し替えまたは注釈が必要
- topics の `content` は違反ではないが、本文キーワードとの対応が弱い
- 後半の「AIで再利用する手順」が前半の 3 差分と明示的に接続していない
- Instagram の列挙が本文テンプレートの出力対象とズレている

### 推奨アクション

1. **High 1 件（指摘 1）を先に反映**: 参考リンクを本文主題に近い記事へ差し替えるか、関連理由を注釈で補う
2. **Medium 2 件（指摘 2 / 3）を反映**: topics の日本語化、3差分と再利用手順の接続
3. **Low 2 件（指摘 4 / 5）は仕上げで調整**: 3差分の横断表、Instagram のスコープ整合

### SEO観点での改善提案

- タイトルの `SEO記事` / `SNS投稿` / `AI` は本文・topics と整合している
- `content` は本文中にないため、`ライティング` または `コンテンツ制作` へ差し替えるとタグと本文の一貫性が上がる
- 参考リンクは Zenn 内回遊に効くため、同じ「AI記事制作」系の記事へ寄せると関連記事として自然
