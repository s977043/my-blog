### 該当箇所 2
L67-L72（Codex のインストール手順に pin コマンドが混入）

```
codex plugin marketplace add s977043/river-review
codex plugin install river-review@river-review-marketplace
codex plugin marketplace add s977043/river-review@v1.2.1  # バージョン pin する場合
```

### 問題点
Claude Code 側は 3 行（add → install → reload）で揃っているのに対し、Codex 側は 3 行目が「reload 相当」ではなく「pin する場合の marketplace add 再実行」になっている。L56「**導入は 3 ステップ**で済むようになりました」の主張と表面上揃っているように見えるが、実際は Codex は 2 ステップ + 例外の 1 行で、本文の「3 ステップで Claude Code と Codex の流れが揃っている」（L74）と乖離する。技術的にも `plugin marketplace add` を 2 回実行する意図が読み取れない（Webエンジニア視点）。

### 提案
pin の例は本筋から外し、L76 の引用ブロックに統合する。

```bash
codex plugin marketplace add s977043/river-review
codex plugin install river-review@river-review-marketplace
codex plugin reload river-review   # もしくは Codex 側のリロード相当コマンド
```

その上で、L76 の注記を「バージョンを pin したい場合は `codex plugin marketplace add s977043/river-review@v1.2.1` のようにタグを付ける」と統合する。Codex に reload コマンドが存在しないなら、L74 の「**同じ流れが揃っている**」表現を「**ほぼ揃っている（Codex は reload 不要）**」に弱める。

## 総合評価

### 良い点
- 冒頭 TL;DR が 4 項目で価値を先出ししており、Zenn 読者の判断時間が短い
- v0.x → v1.x の比較表（L112-L120）が、設計判断を一覧で示せていて理解効率が高い
- 「Claude Code / Codex の使い分け」表（L99-L104）で、配布側（共有）と利用側（使い分け）の分離が明確
- フォールバックを「廃止せず格上げ」する立場が、OSS 運用論として説得力ある
- セクション要点の 1 行先出しが多くの章で守られている

### 改善点
- 「v0.x はテンプレコピーで手間 → v1.x で解消」という同じ主旨が章をまたいで 3〜4 回反復している
- 「学び」章が本筋章の言い換えに近く、Meta レイヤとしての宣言が弱い
- Codex 側のコマンド 3 行目（pin の独立コマンド）が Claude Code との対称性を崩している
- `.codex-plugin/` の唐突な登場、`interface metadata` の英語放置など、用語整備に余地
- まとめ章にタイトル「テンプレコピーをやめた」と呼応するキーフレーズ再掲がない

### 推奨アクション
1. Codex 章のコマンドブロックを修正（pin は引用ブロックに統合、reload の有無を明示）→ 指摘 2
2. 「学び」章のサブタイトルを「他 OSS に転用するなら」に変えて Meta であることを宣言、各項目に転用条件を 1 文ずつ追記 → 指摘 5
3. 重複表現の整理：v0.x 章末（L46）の段落を削除 or 抽象化、まとめ章にキーフレーズ再掲を入れる → 指摘 6・7
4. `:::message` の役割を「想定読者 + 前提知識」に切り替え、関連記事リンクは末尾に集約 → 指摘 10
5. `Manual Era` / `Plugin Era` / `interface metadata` など英語ラベルに初出時の日本語副題を追加 → 指摘 3・4
6. Codex の plugin マーケットプレイスコマンド体系について、公式ドキュメント参照 or 動作確認バージョンを明記 → 指摘 9
