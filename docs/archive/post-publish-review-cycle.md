## 二地平レビュー（dual-horizon）

技術記事は「初速＝拡散/媒体内評価」と「持続＝SEO/AI検索流入」で時定数が違う。30日だけでは SEO が立ち上がりきらない（Gemini 検証）。2つの地平で見る。

| 地平 | 時期 | 見るもの | 主な判断 |
|---|---|---|---|
| **瞬発（T+30日）** | 公開30日後に1回 | Views / Likes / Comments / 媒体内トレンド / GitHub stars・Issues の初動 | このテーマ/媒体が刺さったか。次クラスタ選定の一次入力 |
| **持続（T+180日）** | 公開約180日後に1回 | GSC の検索クエリ・表示回数・CTR・掲載順位 / 流入の定着 / AI検索（Perplexity 等）での参照有無 | 資産化判定 → **リライト要否**。被リンク/継続流入の有無 |

- 対象は「直近に公開した1テーマ（シリーズ単位）」のみ。全記事は追わない
- T+30 と T+180 はテーマ単位で1回ずつ。常時モニタリングしない

## 計測の最小手段（自作しない）

- **各媒体標準アナリティクス**（Zenn/Qiita/note）: T+30 の瞬発はこれを眺めるだけ
- **Google Search Console 一括エクスポート**: 月1回、対象テーマの検索クエリ/順位をスプレッドシートへ落とすだけ（ダッシュボード自作禁止・GSC が検索クエリ資産の蓄積器）
- **GitHub**: stars / issues / 参照元（リポジトリが課題文脈で参照されるまでラグがある前提で T+180 も見る）

> 指標を増やさない。上記以外は原則見ない。3テーマ回して項目が固定したら、その時点で初めて自動化を検討する。

## リライト判定基準（T+180 で評価）

T+180 レビューで各記事を3分類し、次の四半期バックログに反映する。

| 判定 | 条件の目安 | アクション |
|---|---|---|
| **資産（伸ばす）** | 検索流入が定着・順位上昇傾向・AI検索で参照される | リライト投資対象。鮮度更新・一次洞察(I-message)の追記でSEO/GEOを維持 |
| **様子見** | 流入は弱いが下降でない／テーマ自体は有効 | 次サイクルまで放置。判定保留 |
| **撤退（投資しない）** | 流入ほぼゼロ・検索意図とズレ | 新規リライト投資はしない。学びだけ次テーマ選定へ |

- リライトは「新規執筆を減らしてでも資産記事の鮮度更新を優先」する（Gemini: 鮮度がこれまで以上に評価される）
- リライト時は事実更新だけでなく、AI検索に引用される**一次洞察・意思決定理由・苦労（first-party insight / GEO）**を必ず1つ厚くする

## 次クラスタ還流フロー

```
T+30 瞬発結果 ─┐
               ├─→ 月次振り返り(30-45分) ─→ 次の「1クラスタ × 媒体配分」を決定
T+180 持続結果 ┘                          └─→ 資産記事のリライトをバックログへ
```

- 判断単位は「記事案」ではなく「次の1クラスタ」（content-channel-strategy.md の Topic clusters から1つ選ぶ）
- 還流先: `content-channel-strategy.md` の Topic clusters / 30-day plan を更新（新基盤を作らない）
